# 一、问题引入

使用 **`jsPlumb`** 实现拖拽连接线功能时，当节点数量较多（如 >100 个）或频繁操作，**卡顿、掉帧、内存泄漏** 是常见问题。这是因为 `jsPlumb` 默认在每次拖拽/连接时都会：

- 重绘所有连接线（`Canvas/SVG`）
- 监听大量 `DOM` 事件
- 未做性能隔离

下面从 **原理分析 → 配置优化 → 代码实践 → 替代方案** 全面解决 **`jsPlumb` 拖拽卡顿问题**。

## 1、`jsPlumb` 卡顿的核心原因

| 原因                            | 说明                                                         |
| ------------------------------- | ------------------------------------------------------------ |
| **1. 连接线重绘开销大**         | 每次节点移动，所有连接该节点的线都要重新计算路径并重绘（`SVG/Canvas`） |
| **2. 事件监听器过多**           | 每个 `.jtk-endpoint` 都绑定 `drag`、`click` 等事件，节点多则内存占用高 |
| **3. 未启用硬件加速**           | `SVG` 默认不走 `GPU`，复杂图形渲染慢                         |
| **4. 自动重绘未节流**           | `repaintEverything()` 被频繁调用                             |
| **5. 连接点（`Endpoint`）过多** | 每个节点多个 `endpoint` 会指数级增加计算量                   |
| **6.节点过多导致 `DOM` 树庞大** | 浏览器渲染树更新成本高（`O(n)` 或更高）                      |

> 💡 **关键结论**：
>  **`jsPlumb` 适合中等规模（<100 节点）的流程图，大规模需深度优化或换方案。**

## 2、高性能拖拽核心优化策略

#### 策略 1：**使用 `transform` + `will-change` 启用 `GPU` 加速**

```
.draggable {
  /* 避免使用 top/left */
  transform: translate(0, 0);
  will-change: transform; /* 提示浏览器提前优化 */
}
```

#### 策略 2：**虚拟拖拽代理（`Ghost Element`）**

- 拖拽时**隐藏原节点**，创建一个轻量级**克隆元素**作为视觉反馈
- 原节点位置**仅在 `drop` 时更新一次**

#### 策略 3：**事件节流（`Throttle`）**

```
// 使用 requestAnimationFrame 节流
let ticking = false;
function onDragMove(e) {
  if (!ticking) {
    requestAnimationFrame(() => {
      updateDragPosition(e);
      ticking = false;
    });
    ticking = true;
  }
}
```

####  策略 4：**避免实时重排 —— 使用缓存位置**

- 初始化时**一次性计算所有节点位置**（`getBoundingClientRect()`）
- 拖拽过程中**只比对坐标，不查 DOM**

#### 策略 5：**使用 `CSS` `contain` 限制重排范围（现代浏览器）**

```
.container {
  contain: layout style paint;
}
```

# 二、超多节点

当节点数量 **超过 1 万（10,000+）** 时，**`jsPlumb` 已完全不适用** —— 它的设计初衷是中小型流程图（<100 节点），在万级节点下会出现：

- 内存爆炸（每个节点创建多个 `DOM` + 事件监听器）
- 拖拽卡顿到无法操作（`FPS < 5`）
- 连接线重绘耗时数百毫秒
- 浏览器标签页无响应甚至崩溃

## 正确方案：**放弃 `jsPlumb`，改用高性能图形引擎**

以下是针对 **10,000+ 节点** 场景的 **工业级解决方案**。

##  推荐方案 1：**`AntV X6`（国产开源，首选）**

> `GitHub: https://github.com/antvis/X6`
>  官网: `https://x6.antv.vision`

####  为什么适合万级节点？

| 特性                                | 说明                                        |
| ----------------------------------- | ------------------------------------------- |
| **`Canvas` 渲染**                   | 不依赖 `DOM`，`GPU` 加速，渲染效率高        |
| **虚拟滚动（`Virtual Scrolling`）** | 只渲染可视区域节点（如 100 个），其余不创建 |
| **`Web Worker` 支持**               | 布局计算移出主线程                          |
| **增量渲染**                        | 拖拽/缩放时只更新变化部分                   |
| **`TypeScript + Vue/React` 友好**   | 完善的工程化支持                            |

#### 💡 示例：`X6` 虚拟滚动 + `10k` 节点

```
import { Graph, Node } from '@antv/x6'

// 1. 创建画布
const graph = new Graph({
  container: document.getElementById('container')!,
  width: 1200,
  height: 800,
  // 启用虚拟渲染（关键！）
  virtual: true,
  // 关闭不必要的交互
  interacting: false,
})

// 2. 生成 10,000 个节点数据（不立即渲染）
const nodes: Node.Metadata[] = []
for (let i = 0; i < 10000; i++) {
  nodes.push({
    id: `node-${i}`,
    shape: 'rect',
    x: (i % 100) * 120,
    y: Math.floor(i / 100) * 60,
    width: 100,
    height: 40,
    label: `Node ${i}`,
  })
}

// 3. 批量添加（X6 内部自动虚拟化）
graph.addNodes(nodes)

// 4. 启用拖拽（高性能）
graph.enablePanning() // 平移
graph.enableRubberband() // 框选

// 5. 按需连接（避免全连）
// graph.connect({ source: 'node-0', target: 'node-1' })
```

> ✅ 效果：
>
> - 内存占用 < `100MB`
> - 拖拽/缩放流畅（`60 FPS`）
> - 首屏加载 < `1s`

##  推荐方案 3：**自研（`Three.js + Force Layout`）**

适用于 **关系网络、知识图谱** 等 `3D/2D` 场景。

#### 技术栈：

#### 2. 组件代码（`GraphEditor.vue`）

```
<template>
  <div class="graph-container">
    <!-- 工具栏 -->
    <div class="toolbar">
      <button @click="addRandomNode">➕ 添加节点</button>
      <button @click="connectRandom">🔗 随机连线</button>
      <button @click="removeSelected">🗑️ 删除选中</button>
      <span>节点数: {{ nodeCount }}</span>
    </div>

    <!-- 画布容器 -->
    <div ref="graphContainer" class="graph"></div>
  </div>
</template>

<script setup>
import { onMounted, ref, onBeforeUnmount } from 'vue'
import { Graph, Shape, ToolsView } from '@antv/x6'

const graphContainer = ref(null)
const graph = ref(null)
let nodeIdCounter = 10000 // 从 10000 开始，避免冲突

// 初始化
onMounted(() => {
  graph.value = new Graph({
    container: graphContainer.value,
    width: '100%',
    height: '100%',
    
    // ✅ 关键：启用虚拟化（只渲染可视区域）
    virtual: true,
    
    // 交互配置
    panning: true,           // 拖动画布
    mouseWheel: { enabled: true, modifiers: 'ctrl' }, // ctrl+滚轮缩放
    selecting: { enabled: true, multiple: true },     // 框选
    
    // 节点默认样式
    node: {
      width: 120,
      height: 40,
      label: 'Node',
      attrs: {
        body: { fill: '#fff', stroke: '#333', strokeWidth: 1 },
        label: { fill: '#333', fontSize: 12 }
      }
    },
    
    // 连接线样式
    edge: {
      shape: 'edge',
      router: 'manhattan', // 正交连线
      connector: 'rounded', // 圆角
      attrs: {
        line: { stroke: '#666', strokeWidth: 2 }
      }
    }
  })

  // 启用历史记录（支持 undo/redo）
  graph.value.use(new Graph.History())

  // 添加 10,000 个初始节点（分布在一个大网格上）
  const initialNodes = []
  for (let i = 0; i < 10000; i++) {
    initialNodes.push({
      id: `node-${i}`,
      x: (i % 200) * 150,   // 200 列
      y: Math.floor(i / 200) * 80, // 50 行
      label: `Node ${i}`
    })
  }
  graph.value.addNodes(initialNodes)

  // 监听选择变化
  graph.value.on('selection:changed', ({ added, removed }) => {
    console.log('选中节点:', added.map(n => n.id))
  })
})

// 计算当前节点数
const nodeCount = computed(() => {
  return graph.value?.getNodes().length || 0
})

// 新增节点
function addRandomNode() {
  const x = Math.random() * 20000 // 大画布
  const y = Math.random() * 10000
  const id = `node-${nodeIdCounter++}`
  graph.value.addNode({
    id,
    x,
    y,
    label: `New Node ${id}`
  })
}

// 随机连线（从选中节点出发）
function connectRandom() {
  const selected = graph.value.getSelectedCells().filter(cell => cell.isNode())
  if (selected.length === 0) {
    alert('请先选中一个节点')
    return
  }
  
  const source = selected[0]
  const allNodes = graph.value.getNodes()
  const target = allNodes[Math.floor(Math.random() * allNodes.length)]
  
  if (source.id !== target.id) {
    graph.value.addEdge({
      source: source.id,
      target: target.id
    })
  }
}

// 删除选中项
function removeSelected() {
  const selected = graph.value.getSelectedCells()
  selected.forEach(cell => {
    graph.value.removeCell(cell.id)
  })
}

// 销毁
onBeforeUnmount(() => {
  graph.value?.dispose()
})
</script>

}
</style>

## 关键性能保障措施（`X6` 内部已实现）

| 技术                            | 说明                                                    |
| ------------------------------- | ------------------------------------------------------- |
| **`Virtual Scrolling`**         | 只创建可视区域内的 `DOM/Canva`s 对象，内存恒定          |
| **`Incremental Rendering`**     | 拖拽/缩放时只重绘变化部分                               |
| **`Object Pooling`**            | 节点对象复用，减少 `GC`                                 |
| **`Batch Update`**              | `graph.batchUpdate(() => { ... })` 批量操作避免多次重绘 |
| **`Web Worker Layout`（可选）** | 复杂布局计算移出主线程                                  |
