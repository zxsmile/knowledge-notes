## 一、`setup`选项的写法和执行时机

1、`setup`选项的执行时机？

- `beforeCreate`钩子之前，自动执行

2、`setup`写代码的特点是什么？

- 定义数据+函数，然后以对象方式`return`

3、<script setup>解决了什么问题？

- 经过语法糖的封装更简单的使用组合式`API`

4、`setup`中的`this`还指向组件实例吗？

-  指向`undefined`

```

<template>
 <div>{{ name }}</div>
 <button @click="changeName">修改名字</button>
</template>

<script  lang="ts">
import { ref } from 'vue';
  export default{
    name: 'Preson',
    setup(){
       let name = ref('张三');
       let changeName = () => {
          name.value = '李四';
       }
       return {
        name,
        changeName
       }
    }
  }
</script>



<style scoped>
</style>
```

5、`setup`的返回值也可以是一个渲染函数

```
<template>
 <div>{{ name }}</div>
 <button @click="changeName">修改名字</button>
</template>

<script  lang="ts">
import { ref } from 'vue';
  export default{
    name: 'Preson',
    setup(){
       let name = ref('张三');
       let changeName = () => {
         name.value = '李四';
       }
       return () => 'haha' //页面会直接渲染haha，不会管template
    }
  }
</script>
```

6、`vue2`的选项式`api`和`vue3`的组合式`api`可以同时存在，并且`vue2`可以读取`vue3`的内容，`vue3`不能读取`vue2`的内容。

```

<template>
 <div>{{ name }}</div>
 <button @click="changeName">修改名字</button>
 <div>{{ s }}</div>
</template>

<script  lang="ts">
import { ref } from 'vue';
  export default{
    name: 'Preson',
    data(){
        return {
            A: 'uuu',
            s:this.name //vue3的数据
        }
    },
    methods:{
        changeNameA(){
            this.A = '李四';
        }
    },
    setup(){
       let name = ref('张三');
       let changeName = () => {
         name.value = '李四';
       }
       return {
        name,
        changeName
       }
    }
  }
</script>
```
