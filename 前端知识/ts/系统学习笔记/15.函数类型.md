# 一、函数类型

## 1、常规参数类型

在函数形式参数中，为参数添加类型注解基于能定义参数的类型

默认情况下，我们不屑任何类型时，参数会隐式添加上any类型

开启`noImplicitAny`开启之后，编辑器会提示不可以有隐式的`any`，要想设置为`any`需要显示设置。

```
function add(x:number,y:number){
  return x+y
}
//函数表达式和匿名函数，我们也可以用相同的方法来被定义
const f = function(x:number,y:number){
  return x+y
}

//如果我们没有指明类型，会隐式添加any类型
function push(x,y){
   return x+y
}
```

## 2、可选参数类型

TS中，默认情况下，每一个参数都是必选参数，在调用函数时，编译器会检查传入的参数的个数和函数定义时形参的个数是否一致，不相等产生编译错误。

如果一个参数设置为可选参数，那就可以不传，可选参数的设置同样时参数名后添加一个"?"。

可以定义多个可选参数，但是必选参数必须位于可选参数之前

可选参数可以不传

设置为可选参数后，该值可以传设置的类型值或者undefined

```
function fn(x:number ,y?:number,z?number){
   return x + (y??0)
}
fn(1)
fn(1,2)
fn(1,undefined)
```

## 3、默认参数类型

3.1、函数默认参数类型可以通过类型注解定义，也可以根据默认参数值自动推断类型

```
function add(x:number = 1,y=2){
  reurn x + y;
}
```

3.2、如果函数定义了默认参数，并且参数是在必选参数之前，这个时候默认参数就变成了可选参数

```
function add(x:number = 1,y=2,z=3){
  reurn x + y;
}
add(3)
add(1,6)
y和z都为可选参数
```

3.3、在语法上，同一个函数参数不允许同时声明为可选和默认的，否则会报错

```
function add(x:number = 1,y?=2 //报错){
  reurn x + y;
}
y同时声明为可选和默认,会报错
```

3.4、如果默认参数在必选参数之前的话，那它依旧是个必选参数

```
function add(x:number = 1,y：number){
  reurn x + y;
}
//x为默认参数，y为必选参数，默认参数不在末尾，为必选参数
```

## 4、剩余参数类型

处理多个参数，如果函数定义中声明了剩余参数，那么在调用时，会将多余的实际参数都收集到剩余参数列表中。因此，分为数组类型或者元组类型。

#### 数组的剩余参数

最常见的就是将剩余参数声明为数组类型

在调用时可以接受多个也可以是0个

```
function f(...args:number[]){}
f()
f(1,2,3)
f(1)
```

#### 元组的剩余参数

最常见的就是将剩余参数声明为元组类型

```
//常规元组类型
function f(...args:[boolean,number]){}
相当于===>
function f(args0:boolean,args1:number){}
f(true,0)
```

```
//可选元组类型
function f(...args:[boolean,number?]){}
f(true,0)
f(true)
```

```
//带有剩余元素的元组类型
function f(...args:[boolean,...number[]]){}
f(true)
f(true,10,20,10)
```

## 5、解构参数类型

解构可以应用在函数参数列表中

```
function fn([x,y]){}
fn([1,2])
function fn({x,y}){}
fn({x:3,y:6})
```

还可以使用类型注解为解构参数添加类型信息

```
function fn([x,y]:[number,string]){}
fn([1,'333'])
function fn({x,y}:{x:number,y:string}){}
fn({x:3,y:6})
```

## 6、函数返回值类型

1、可以自己手动添加返回值类型，`ts`也会根据函数体内的返回值自动推断出函数返回值类型。

```
function fn(x:number):number(){
   return x
}
```

2、`void`空类型，该类型唯一有意义的使用场景就是作用函数返回值类型。如果一个哈部署返回值设置了`void`类型，这个它只能返回`undefined`或者没有返回值。

```
function fn(x:number):void(){
   return undefined
}
function fn(x:number):void(){
   
}
```

## 7、函数类型字面量

函数类型字面量是定义函数类型的方法之一，它能够指定函数的参数类型、返回值类型以及泛型类型参数。函数类型字面量的语法：

```
(ParameterList) => Type

ParameterList：参数列表

Type：函数返回值类型
```

```
let f:() => void
f = function() {}
```

1、定义函数参数时的形参和实际函数的形参不一定非得一样

```
let add:(x:number,y:number)=>number
add = function(z,d){
   return z+d
}
```

2、定义参数类型时，必须是参数名：参数类型

3、必须明确指定返回值类型，如果没有返回值就写`void`

## 8、调用签名

函数本质上是一个对象，但是特殊在函数是可调用的对象。因此，可以使用对象类型来表示函数类型。若在对象类型中定义了调用签名类型成员，那么我们就叫该对象是函数类型。

```
{

   (ParameterList)：Type

}

ParameterList：参数列表

Type：函数返回值类型
```

```
let obj:{(x:number):number}
```

单个参数的对象类型字面量的简写形式就与函数类型字面量的单个形式相同

```
let obj:(x:number)=>number
相当于
let obj:{(x:number):number}

let abs:(x:number)=>number = Math.abs
相当于
let abs:{(x:number):number }= Math.abs
```

对象类型字面量具有更强的类型表达能力，函数类型字面量更简洁。

```
function fn(x:number){
  console.log(x)
}
fn.version = '1.0'
let foo:(x:number)=>void = fn
cont version  = foo.version //报错，拿不到version

let foo1:{(x:number):void ,version:string}= fn
const version1  = foo1.version //可以拿到version
```
