# 一、泛型

泛型程序设计是一种编程风格或者编程范式，它允许在程序中定义形式类型参数，然后在泛型实例化时使用实际类型参数来替换形式类型参数。

通过泛型，我们能够定义通用的数据结构或类型，这些数据结构或类型仅在它们操作的实际类型上有差别。

泛型程序设计是实现可重用组件的一种手段。

通过一个`identity`函数来介绍泛型的基本应用。也叫做恒等函数，它的返回值永远等于传入的参数。

```
function identity<T>(arg:T):T{
  return arg
}

const foo = identity<string>('123')
const foo1 = identity<number>(11)
```

# 二、形式类型参数声明

表示绑定到泛型类型或泛型函数调用的某个实际类型。

语法：

```
<TypeParameter,TypeParameter,...>
```

`TypeParameter`代表形式参数名，形式参数类型都需要'`<>'`之间，存在多个时，用','进行隔开。

一般类型参数名以大写字母开头，因为它代表一个类型。编程风格上来说，推荐起一个有描述性的名称：`TResponse`，同时还建议形式类型参数名以大写字母T作为前缀。另一种流行的命名方法是使用单个大写字母作为形式参数的类型名，通常由字母T开头，一次往下，`U、V`等大写字母

语法：

```
function assign<T,U>(target:T, source:U):T & U{
  //...
}
```

# 三、参数默认及可选类型

#### 默认参数

在声明形参时，可以为类型参数设置一个默认类型，类似于函数默认参数。

```
<T=defaultType>
```

在语法中可以使用'='连接

```
<T=boolean>
```

默认类型可以引用前一个

```
<T，U = T>
```

#### 可选类型参数

如果一个形参没有定义默认类型，那么它是必选类型参数。假如说定义了默认类型，那么它就是可选的类型参数。在形参列表中，必选不允许出现在可选之后。

```
<T=boolean, U> //错误
<T, U=boolean>
```

从左到右解析并设置类型参数的默认类型，所以不可以左侧去引用右侧声明的类型参数

```
<T = U, U=boolean> //错误
```

# 四、实际类型参数

在引用泛型类型时，可以传入一个实际类型参数作为形式类型参数的值，该过程称作泛型的实例化。

```
<Type, Type, ...>
```

1、当显式传入实际类型参数时，只有必选类型参数是一定要提供的，可选类型参数可以被省略，这个时候可选类型参数将使用默认类型。

```
function identity<T>(arg:T):T{
  return arg;
}

identity<number>(1)
identity<Date>(new Date())

function f<T, U=boolean>(){}
f<string,number>()
f<string>()
```

# 五、泛型约束声明

语法：

```
<T extends CT>
T:代表形式参数类型参数名
extends:关键字
CT:表示一个类型，该类型用于约束T的可选类型的范围的
```

```
interface Point{
  x:number;
  y:number;
}

function identity<T extends Point>(x:T):T{
  return x
}
identity({x:0,y:0})
identity({x:0,y:0,z:0})
identity({x:0})//报错
```

可以同时定义约束和默认类型

```
function fn<T extends number = 0>(){}
fn<1>()
fn<2>()
fn<'12'>()//报错
```

# 六、泛型约束引用类型参数

```
<T, U extends T>
<T extends U, U>
```

#### 基约束

规则三个：

1、`<T extends U>`类型参数`T`的基约束就是类型参数`U`

2、如果声明了`T`且为具体某一个类型，参数`T`的基约束就是这一类型

`<T extends number>`基约束就是`number`

3、如果`T`没有约束，那么类型参数`T`的基约束就是空对象字面量'`{}`'

# 七、泛型函数定义

若一个函数的哈数签名中带有类型参数，那么它就是一个泛型函数，泛型函数中的类型参数用来描述不同参数之间以及参数和函数返回值之间的关系。

定义调用签名：

```
<T>(x:T):T
T为泛型形式类型参数
```

定义泛型构造签名的语法：

```
new <T>():T[]
T为泛型形式类型参数
```

```
//例一
function f1<T>(x:T):T{
    return x
}
const a:string = f1<string>('111')
const b:number = f1<number>(22)
//返回值类型和参数类型相同即可
```

```
//例二
function f2<T>(x:T,y:T):T[]{
  return [x,y]
})
const c:number[] = f2<number>(0,1)
f2函数接受两个相同参数类型的参数，且返回值是数组且数组中元素类型与参数类型相同
```

```
//例三
function f3<T,U>(x:T,y:U):{x:T,y:U}{
  return {x,y}
}
const d:{x:string,y:number} = f3<string,number>('a',8)
//接受两个不同类型参数，并且返回类型为对象类型，且对象中的属性类型与参数类型相同
```

```
//例四
function f4<T,U>(a:T[],b:(x:T)=>U):U[]{
  return a.map(b)
}
const e:boolean[] = f4<number,boolean>([0,1,2], n=> !!n)
//接受两个参数，第一个参数为任意类型的数组，第二个参数是个函数、
//该函数的参数类型与第一个参数元素类型相同，该函数的返回值是任意类型
//整体f4返回值参数类型与第二个参数函数的类型相同
```

# 九、泛型函数的注意事项

如果一个函数既可以定义为泛型也可以定义非泛型这种情况我们要使用非泛型

比如：函数类型只出现一次

泛型是用来关联多个不同值的类型的，或者关联参数与返回值的关系

# 十、泛型接口

若接口的定义中带有类型参数，那么它就是泛型接口。在泛型接口定义中，形参列表紧随接口名之后

语法：

```
interface MyArray<T> extends Array<T>{
  first:T |undefined,
  last:T | undefined,
  ...
}
引用泛型接口时，必须指定实际类型参数，除非类型参数定义默认类型。
```

```
const a:Array<number> = [1,2,3]
const b:number[] = [1,2,3]
```

# 十一、泛型类型别名

类型别名的定义中带有类型参数，它就是泛型类型别名

```
type Name <T> = T | undefined | null
```

在引用泛型类型别名表示的类型时，必须指定实际类型参数

```
type Container<T> = {value:T}
const a:Container<number> = {value:0}
const v:Container<string> = {value:'1'}

type Tree<T> = {
  value:T,
  left:Tree<T> | null,
  right:Tree<T> | null
}

const tree:Tree<number> = {
  value:0,
  left:{
    value:12,
    left:null,
    right:null
  },
  right{
    value:12,
    left:null,
    right:null
  }
}
```

# 十二、泛型类

若类的定义中带有类型参数，那么它就是泛型类

不管是类表达式还是类声明，都可以使用泛型

```
class Container<T>{
  constructor(x:T){
  
  }
}

const a = new Container<boolean>(true)
const b = new Container<string>('123')

const Container = class <T>{
  constructor(x:T){
  
  }
}

const a = new Container<boolean>(true)
const b = new Container<string>('123')
```

```
interface A<T>{
  a:T
}
class Base<T>{
  b?:T
}

class B<T> extends Base<T>{
  constructor(x:T){
    super()
  }
}

class C<T> imlements A<T>{
  a:T
  constructor(x:T){
  }
}
```

静态类型成员不可以定义为泛型

```
interface A<T>{
  a:T
}
class Base<T>{
  b?:T
}

class B<T> extends Base<T>{
  static a:T //错误
  constructor(x:T){
    super()
  }
}
```
