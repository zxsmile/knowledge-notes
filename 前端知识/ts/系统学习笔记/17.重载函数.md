# 一、重载函数

是指一个函数同时拥有多个同类的函数签名。当使用不同数量和把类型的参数调用重载函数时，可以执行不同的函数实现代码

**注意：多个且同类 ===>调用签名两个及以上或者构造签名两个及以上。**

重载函数的定义由两部分组成：

- 一条或者多条函数重载语句

- 一条函数实现语句

```
//函数实现语句
function add (x:number|any[],y:number|any[]):any{
   if(typeof x === 'number' && typeof y === 'number' ){
     return x + y
   }
   if(Array.isArray(x) && Array.isArray(y) ){
     return [...x,...y]
   }
}

//函数重载语句
const a:number = add(1,2)
const b:any [] = add([1],[2])
```

## 函数重载

1、不带有函数体的声明语句叫做函数重载；不包含函数体，它只提供函数的类型信息。

```
//函数重载
function add(x:number,y:number):number
function add(x:any[],y:any[]):any[]
```

2、函数重载只在编译阶段，在编译生成`js`后会被删除，因此在最终生成的`js`代码是不包含函数重载的代码。

3、函数重载允许一个或者多个，但是只有多个时才有意义。因此只有一个时，就直接定义就好了。

4、函数重载应该位于函数实现之前，并且每一个函数重载的函数名和函数实现中的函数名必须一致。

5、函数重载与函数实现语句之间不能存在其他的语句，否则就会产生编译错误。

```
//函数重载
function add(x:number,y:number):number
function add(x:any[],y:any[]):any[]
console.log(222) //编译错误
//函数实现
function add (x:number|any[],y:number|any[]):any{
   if(typeof x === 'number' && typeof y === 'number' ){
     return x + y
   }
   if(Array.isArray(x) && Array.isArray(y) ){
     return [...x,...y]
   }
}

```

## 函数实现

1、函数实现包含了实际函数体代码，并且在编译时存在，并且在编译后生成的`js`中也同样存在。每一个重载函数只允许有一个函数实现，它只能位于所有函数重载语句之后，否则会产生编译错误。

2、我们在函数重载中的函数签名是每一个函数重载中独立存在的，也就是说每个函数重载都有一个调用签名，但是函数实现中的函数签名不属于重载函数的调用签名之一的，所以不可以混用。

 3、函数实现需要兼容每个函数重载中的函数签名的。

4、需要开发者自行去判断类型做兼容

```
//函数重载
function add(x:number,y:number):number
function add(x:any[],y:any[]):any[]

//函数实现
function add (x:number|any[],y:number|any[]):any{
   if(typeof x === 'number' && typeof y === 'number' ){
     return x + y
   }
   if(Array.isArray(x) && Array.isArray(y) ){
     return [...x,...y]
   }
}
add(1,2) //正确
add([1],[2])//正确
add([1],2)//错误
```

## 函数重载的执行顺序

当程序中调用了一个重载函数时，编译器首先构建出一个候选函数重载列表。

一个函数重载只有满足以下条件才能成为本次函数调用的候选哈纳湖重载

1、函数实际参数的数量不少于函数重载中定义的必选参数的数量

2、函数实际参数的数量不多于函数重载中定义的必选参数的数量

3、每个实际参数的类型能够赋值给函数重载定义中对应形式参数的类型

**总结：必选参数数量相等，类型相等**

```
function f(x:string):void; //重载1
function f(y:'specialized'):void; //重载2

function f(x:string){
  console.log('123')
}//函数实现

//候选列表
//function f(y:'specialized'):void; //重载2 ===>字面量类型，所以优先级要高于重载1
//function f(x:string):void; //重载1 


```

#### 解析顺序：

1、字面量类型的优先级要比普通的高

2、当我们实际调用的参数没有在候选函数列表中时，会产生编译错误

注意：因为ts没有很只能，它只能把候选函数重载列表的首位当作它最终使用的一个函数重载，所以我门需要注意函数重载时的顺序问题

```
function f(x:any):number; //重载1
function f(x:string):0|1; //重载2

function f(x:any):any{
  console.log(x)
}//函数实现

const a:0|1 = f('hi') //编译错误
这里是因为，函数重载是根据函数参数判断执行那个的，上面两个函数重载的参数都满足时('hi'既满足any又满足string)，函数重载的顺序就显得尤为重要了，这个时候会选择函数重载列表的首位当作它最终使用的一个函数重载，这个例子中，由于函数重载列表的首位为重载1，所以返回值0|1和number不对应,报错

function f(x:'string'):0|1; //重载2
function f(x:any):number; //重载1


function f(x:any):any{
  console.log(x)
}//函数实现

const a:0|1 = f('hi') //编译正确
const b = f('hi') //编译正确

```

## 重载函数的类型

函数实现的函数签名不属于重载函数的调用签名之一

调用签名的书写顺序是有意义的，它决定了函数重载的解析顺序

```
function f(x:string):void; //重载1
function f(x:number):number; //重载2

function f(x:any):any{
  console.log(x)
}//函数实现

相当于 对象类型字面量
{
  (x:string):void
  (x:number):number
}
```

## 函数中的`this`的类型

`this`是`js`中的关键字，它可以表示调用函数的对象或者实例对象等

在默认情况下，`this`的类型是`any`，并允许程序在`this`值上执行任意的操作。

因为编译器不会对`any`类型进行类型检查

## `this`类型设置

ts中是允许this作为函数的参数来定义它的类型

#### `this`参数

1、`this`必须在参数的首位

2、`this`参数类型只在编译阶段，不会再编译生成的`js`中存在

3、`this`类型为`void`时，在函数体中就不允许读写`this`的属性和方法

4、在定义了`this`的类型之后，我们就不能直接去执行，要用`Function.prototype.call`方法去执行，这种方法它能够指定`this`的值。

```
function foo(this: {name:string}):number{
   this.name = 'aa'
   return 1
}
foo()//编译错误
foo.call({name:'xxx'})//编译正确
```
