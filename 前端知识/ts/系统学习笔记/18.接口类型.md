# 一、接口类型

## 接口声明

它能够去表示任意的对象类型，不同的是既可以指定类型参数又可以给对象类型命名。

语法：

```
interface interfaceName{

   TypeMember,

   TypeMember

   ...

}
```

```
interface IMap{
   localtionName: string,
   locationY: number
}
```

## 接口类型的类型成员分类：

#### 属性签名

#### 调用签名

- 定义了该对象类型表示的函数在调用时的类型参数，参数列表以及返回值类型。

- 语法：

  ```
  {
  
     (ParameterList)：Type
  
  }
  
  ParameterList：参数列表
  
  Type：函数返回值类型
  ```

  ```
  interface IMap{
     (message?:string,num?:number):Error
  }
  ```

#### 构造签名

- 定义了该对象类型表示的构造函数在使用`new`运算符调用时的参数列表以及返回值类型

- 语法：

  ```
  {
  
    new (ParameterList)：Type
  
  }
  
  ParameterList：参数列表
  
  Type：构造函数返回值类型
  ```

  ```
  interface IMap{
     new (message?:string,num?:number):Error
  }
  ```

#### 方法签名

- 是声明函数类型的属性成员的简写

- 语法：

  ```
  PropertyName(ParamList):Type
  PropertyName:方法名称，可以为标签符、字符串、数字和可计算属性名
  ParamList：表示可选方法形式参数列表类型
  Type：可选的方法返回值类型
  
  相当于在调用签名前面加了一个方法名称PropertyName	
  ```

  ```
  interface IMap{
     getElementById (elementId:string):HTMLElement | null
  }
  ```

  

- 之所以说它是声明函数类型的属性成员的简写，是因为方法签名可以改写为语法稍微复杂的属性签名。

  ```
  PropertyName:{(ParamList):Type}
  ```

  ```
  interface IMap{
    f(x:string):string //方法签名
    f:{(x:string):string} //属性签名+对象类型字面量
  }
  ```

  

- 在改写的语法中，属性名保持不变并使用对象类型字面量和调用签名来表示函数类型，因为该对象类型字面量中只有一个调用签名，因此可以再次使用函数类型字面量来代替对象类型字面量

  ```
  PropertyName:(ParamList) => Type
  ```

  ```
  interface IMap{
    f(x:string):string//方法签名
    f:(x:string)=>string//属性签名+函数类型字面量
  }
  ```

- 若接口中包含多个名字相同但参数列表不同的方法签名成员，则表示该方法是重载方法，它具有多种调用签名。

  ```
  interface IMap{
    f(x:string):string
    f():number
    f(x:boolean):boolean
  }
  ```

  

  #### 索引签名

  - `js`支持使用索引去访问对象的属性的，即通过"[]"语法去访问对象的属性。

    ```
    const colors = ['red','green','blue']
    const red = colors[0]
    ```

  - 接口中的索引签名能够描述使用索引访问的对象属性类型

  - ##### 字符串索引签名

    ```
    [indexName:string]:Type
    indexName:索引名称，它可以为任意合法的标识符，索引名只是占位，它不代表真实索引名称。在字符串索引签名中，索引名的类型必须是string类型，Type就代表着索引值的类型。
    ```

    

    ```
    interface A{
     [prop:string]:number; //只能是键名为string，键值为number
     a:number,
     b:0,
     c:1|2，
     d:'555' //报错
    }
    
  //使用
    let a:A = {
      a:1,
      b:0,
      c:1
    }
    ```
  
  - ##### 数值索引签名
  
    ```
    [indexname: number]: Type
    indexname也是索引名，它可以为任意合法的标识符，索引名也是只起到占位的作用，它不代表真实的索引名。在数值索引签名中，索引名的类型只能是number，Type表示索引值的类型，它可以为任意类型。
    ```
  
    ```
    interface B{
      [prop:number]:boolean; //只能是键名为number，键值为boolean
      a:true,
      b:false,
      c:boolean
     } 
    
     let b:B = {
      a:true,
      b:false,
      c:false
     }
    ```
  
    注意：当出现同名的一个字符产索引签名及数值索引签名时，数值索引签名必须是可以赋值给字符串索引签名的类型，也可以说数值索引签名类型是字符串索引签名的子集。
  
    ```
    interface B{
      [prop:string]:number;
      [prop:number]:number;
      a:1,
      b:number,
      c:number
     } 
     
     interface B{
      [prop:string]:number;
      [prop:number]:boolean; //编译报错，因为数值索引签名和字符串索引                          //签名的值类型不一样，数值索引签名不是字                         //符串索引签名的子集
      a:1,
      b:number,
      c:number
     } 
     
     
    ```
