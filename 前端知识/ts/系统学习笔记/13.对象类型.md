# 一、对象类型

#### `object`方式-类型兼容性

准确地表示非原始类型，原始类型不允许赋值给`object`类型

#### 顶端类型:

`any`、`unknow`

`object`也可以赋值给`any`和`unknow`

`Object`类型是描述所有对象共享的属性和方法，所以也可以进行赋值

空对象类型字面量:{}，也可以进行赋值

```
const a:0bject = 'h1'
let obj: object
obj = Symbol()
obj = new| Date() 
const a:any = obj;
const b:unkown = obj; 
const obj1:0bject = obj;
const obj2:{} = obj;
```

## 对象定义类型-对象类型字面量

`0bject`:用来描述所有对象共享的属性和方法、自动封箱、

`object`:就是用来表示原始数据类型，不可以对自定义类型进行操作。

`{ x:number, y :number }` ===> 对象类型字面量。

```
const point: {x:number,y :number} ={x:1,y:4} 
point.x
```

#### 对象类型字面量-基本语法

在定义对象类型字面量时，需要将类型成员依次列出

```
{

TypeMember; 

TypeMember; 

TypeMember;

}
```

在此，`TypeMember`表示对象类型字面量中的类型成员，放在`{}`中，且各种类型成员之间，可以使用`";"`号去隔开，也可以使用`","`去隔开,这两种分隔符不存在功能上的差异。

#### 对象类型字面量的类型成员可分成五类:

- 属性签名
- 调用签名
- 构造签名
- 方法签名
- 索引签名

```
const point: {x:number;y:number} 
const point1: {x:number,y:number}
```

这里只讲一个属性签名，后面几个在函数类型中讲

## 对象字面量类型-属性签名

属性签名中声明了对象类型中的属性成员的名称和类型。

```
{
PropertypeName: Type
}
```

`PropertypeName`:对象属性名，可以为标识符、字符串、数字和可计算属性名，`Type`就表示属性的类型。

#### 可计算属性名?

在属性签名中，`Type`也就类型部分是可以省略的，允许只列出属性名称而不定义任何类型，这种情况会默认为`any`类型。

```
 const a:'a' = 'a'
 const b:0 = 0
 
let obj:{
    [a]: boolean;
    [b]: boolean
}

const s: unique symbol = Symbol()

let obj: {
 [s]: boolean;
}

const point:{x,y}相当于
===>
const point:{x: any,y:.any}
```

## 对象类型字面量-可选属性

在默认情况下，通过属性签名定义的对象属性是必选属性。如果我们想要把它变成可选属性，就是属性名后加个`"?"`即可

```
{
typeName?: Type;

}
```

在给对象类型赋值时，可选属性可以被忽略。

在`--strictNullChecks`为`true`的模式下。`TS`会自动在属性的类型定义中添加`undefined`类型，但是不允许`null`进行赋值的，

```
let point:{x:number,y:number,z?:number};
point	{x:1,y:0}	
point	{x:1,y:0,z:0}	//编译正确
point	{x:1,y:0,z:undefined}	//编译正确
point	{x:1,y:0,z:null}	//编译失败
point.x //编译正确
point.y //编译正确
point.z //编译正确
point.t //编译失败
```

## 对象类型字面量-只读属性

属性签名中定义只读属性。添加`readonly`修饰符能够定义只读属性

```
{

readonly typeName: Type; 

readonly typeName: Type;

}

```

那只读属性只可以在初始化时添加，后期不允许被修改 

```
point = {x:10,y: 20} //编译正确的 

point.x = 20; // 编译失败 

point.y = 40; // 编译失败
```

```
let point:{
readonly x:number; 
readonly y: number;
point = {x: 10，y: 20}//编译正确的 
point = {x: 5， y:10}//编译正确的 
point.x	20	// 编译失败
point.y=10  // 编译失败
```

## 对象类型字面量-空对象类型字面量

如果对象类型字面量中没有定义任何类型成员，那么它就变成了一种特殊的类型，即空对象类型字面量`"{}"`。

它表示不带有任何属性的对象类型，因此不允许在`"{}"`上访问任何自定义属性。

```
point.x< ===> 编译错误
可以访问公共的方法
point.valueOf() ==> 编译正确
```

和我们的Object在行为上是一样的，当然它俩就可以互换使用。主要区别在于语义上。

```
let point:{}	
{x:1,y:2}	
point.x
point.value0f() 
```

## 多余属性

对象的多余属性就可以理解为多出来的属性。多余属性是一个相对的概念，只有在比较两个对象类型的关系时说多余属性才有意义。

比如源对象类型和目标对象类型？

```
let point:{x:number, y:number} = {x:1,y:4}
```

所以我们把"="左侧叫做目标对象类型，右侧叫做源对象类型。

所以，当源对象类型中存在一个或多个在目标对象类型中不存在的属性时，我们就说源对象类型和目标对象类型存在多余属性。

```
let point:{x:number, y:number} = {x:1,y:4}
//不存在多余属性
let point:{x:number, y:number} = {x:1,y:4,z:20}
//存在多余属性z
```

#### 编译器对多余属性的检查

会影响类型见的子类型的兼容性以及赋值兼容性，也就是说编译器不允许在一些的操作中存在多余属性。

##### 体现方面：

1、直接赋值

2、作为函数参数来调用时

```
//直接赋值
let ponit:{
    x:number,
    y:number
} = {x:1,y:2,z:0}

//作用函数参数来调用时
function fn(point:{x:number,y:number}){
	console.log(point.x)
}
fn({x:1,y:2,z:0})
```

##### 为什么会存在？

1、误操作

2、在编译阶段能够带来的最直接的帮助发现属性名的拼写错误，并且还根据算法来推测可能的属性名。

```
const task:{cancel:boolean} = {canceln:true}
```

#### 如何允许多于属性

1、类型断言

- 处于类型断言表达式，编译器不再对其进行多于属性检查

```
const p1:{x:number} = {x:0,y:0} as {x:number}
```

2、使用'//@ts-ignore'注释指令

```
//@ts-ignore
const p1:{x:number} = {x:0,y:0}
```

3、添加索引签名

```
const p1:{x:number,[prop:string]:number} = {x:0,y:0,z:4}
```

4、赋值

```
const temp = {x:0,y:0}
const p3:{x:number} = temp
```
