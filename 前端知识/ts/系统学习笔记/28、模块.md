# 模块

它强调将程序按照功能划分为独立可交互的模块。一个模块就是一段可以重新使用的代码，将功能的实现封装在模块内部。一个模块可以声明对其它模块的依赖，且模块之间也可以通过模块的公共`API`进行交互。

作用：提供更好的封装性和可重用性。 

## `CommonJS`

是一个主要用于服务器端`JS`程序的模块系。使用`require`语句来声明对其他模块的依赖，同时使用`exports`语句来到处当前模块内的声明。

用于`Node.js`程序中

## `AMD`

#### 为什么会有`AMD`：

1、`CommonJS`是在浏览器端`JS`环境中不支持

2、`CommonJS`模块系统采用的是同步的方式加载模块文件。阻塞用户操作，用户体验不好

#### 含义：

异步 模块定义。`define`函数来注册一个模块。也提供`require`函数用来声明依赖，还提供`export`语句用来导出当前模块内的声明。

## `UMD`

既可以在浏览器中使用，又可以在`nodeJS`中使用，`UMD`模块基于`AMD`模块的定义，并且针对`CommonJS`模块定义进行适配。相对比较复杂

## `ESM`

能够在任何`JS`运行环境中使用，比如服务端环境还是浏览器环境

`ESM`使用`import`和`export`关键字进行定义

## `TS`中的`ESM`

1、每个模块都拥有独立的模块作用域。通过`import`来声明对其他模块的依赖。同时，通过`export`语句加那个模块内的声明进行公共给其它的模块使用。

2、以文件为单位。一个模块对应一个文件同时一个文件也表示一个模块，两者是一对一的关系。

3、若一个`ts`文件中有`import`或者`export`语句，就是一个模块'`module`'，若一个`ts`文件中没有`import`或者`export`，称为脚本'`Script`'。模块的作用域是当前模块，脚本的作用域是全局，所以模块可以访问外层的全局作用域。

## 模块导出的分类

默认情况下，在模块内部的声明不可以给模块外部访问。若想把模块内部的声明开放给模块外部使用，就需要配合导出语句将模块内的声明进行导出。

#### 模块导出的分类：

##### 命名模块导出

使用自定义的标示符名来区分导出声明。在一个模块中，可以同时存在多个命名模块的导出。使用关键字`export`即可。

```
export let a = 0
export let b = 0
```

###### 命名模块导出列表

```
const a = 0;
const b = 2
export {a, b}
```

##### 默认模块导出

一个模块只允许存在一个默认模块导出，使用的是`export default` 关键字来进行定义

```
export default function f() {}
```

## 聚合模块

是指将其他模块的模块在本模块导出

```
export{a, b} from './mod.ts'
```

## 模块导入

对于一个模块的命名模块的导出，可以通过其导出的名称来导入它。

```
import {a} from './index'
```

导入整个模块

```
import * as utils from './index'
```

导入默认模块

```
import modDefault from './index'
```

`modDefault` 可以为任意标识符，在当前模块中，将使用`modDefault`这个名字来访问`index`模块中的默认模块导出。

## 重命名模块导入

```
import {oldName as newName} from './index'
```

## 针对类型的模块导入和导出

```
//utils.ts
export const a = 0
export interface point{
  x:number;
  y:number;
}
```

```
//index.ts
import {point, a} from './utils.ts'
console.log(a)
const p:point = {
  x:1,
  y:10
}
```

## 导入和导出类型

导入类型时和之前导入模块是一样的，但是导出时需要添加`type`或者`interface`关键字

```
export type {...}

export type {...} from '...'

import tyoe defaultType from ...
```
