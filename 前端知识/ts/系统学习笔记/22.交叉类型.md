# 交叉类型

表示一个值同时属于多种类型，交叉类型通过交叉类型字面量来定义。

定义方法：'&' 符号分隔

若一个值既是类型A又是类型B，那么我们就说该值的类型是交叉类型：A&B

```
interface A{
  click():void;
}
interface B{
  focus():void;
}
type T = A & B;
```

## 1、交叉类型成员类型的运算

1.1、 如果交叉类型中存在多个相同的成员类型，那么相同的成员类型将合并为单一成员类型。

```
type T0 = boolean
type T1 = boolean & boolean
type T2 = boolean & boolean & boolean

相当于===>
type T = boolean
```

1.2、绝大部分情况下，成员类型满足类似于数学中的'加法交换律'，即改变成员类型的顺序不影响交叉类型的结果。

```
interface ClickF{
   click():void
}
interface FocusF{
   focus():void
}
type T0 = ClickF & FocusF;
type T1 = FocusF & ClickF;
//T0和T1是同类型的
```

1.3、交叉类型涉及到调用签名重载或构造签名重载时，会和顺序有关系

```
interface F1{
  register(s:any):void
}
interface F2{
  register(s:string):boolean
}

type C1 = F1 & F2
type C2 = F2 & F1

function foo(f1:C1, f2:C2){
  let a:void  = f1.register(123)
  let b:boolean  = f2.register('22')
}
```

## 2、交叉类型的类型成员

2.1、只要交叉类型中任意一个成员类型包含了属性签名M，那么交叉类型也包含属性签名M。

```
interface A{
  x:{a:boolean};
}

interface B{
  x:{b:boolean};
}

let c:A & B 
===>相当于
x:{
  a:boolean,
  b:boolean
}
```

2.2、当交叉类型的属性签名中都是可选才是可选，有一个是必选的，那属性签名也是必选的

```
interface A{
  x:boolean;
  y?:string;
}

interface B{
  x?:boolean;
  y?:string;
}

let c:A & B = {x:true}
```

## 3、交叉类型和联合类型的优先级

3.1、同时使用时，&有更高的优先级，当作数学中的'x'，那我们'|'如同数学中的'+'

```
A & B | C & D
===>
A * B + C * D
```

3.2、同函数字面量同时使用时，`&`和`|`优先级要更高，相当于加了()

```
() => bigint | number
===>
() => (bigint | number)

() => bigint & number
===>
() => (bigint & number)
```
