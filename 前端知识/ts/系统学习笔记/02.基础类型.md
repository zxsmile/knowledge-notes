# `strictNullChecks`

`--strictNullChecks`:是否严格的检查`null`和`undefined`类型:

1、默认是`false`

2、设置为`true`之后，`undefined`和`null`只能赋值给它本身的类型

3、`undefined`和`null`是不同两种类型，不能互换使用;

# `void`类型

`void`:表示某个值不存在，用作函数返回值类型。

1、假如一个函数没有返回值，那么我们就可以定义它为`void`类型。

2、使用`void`关键字去定义类型

3、当使用定义类型为`void`时，在`strictNullChecks`开启时，只能返回`undefined`,关闭
时，`undefined`和`null`都可以

# `enum`类型

`enum`:是计算机语言中的一种数据类型。枚举类型，
在`TS`中，`enum`是一种原始类型，它通过关键字`enum`来定义的。分类:根据枚举类型中的成员进行的

## 数值型枚举

1、`number`类型的子类型。它由一组命名的数值常量构成。

2、我们可以使用访问对象属性一样去访问枚举成员。

```
enum Direction{
    Up,// 0
    Down,// 1
    Left, // 2
    Right // 3
}
const Dir: Direction = Direction.Up;
console.log(Direction.Up,Direction.Up,Direction.Left, Direction.Right)
//0 1 2 3
```

3、每个数值型枚举成员都表示一个具体的数字，如果在定义时没有设置，TS会自动计算枚举成功的值，会给第一项设置为0，后面的，则是等于前一个枚举成员的值加1;可以在定义时，为一个或者多个枚举成员设置初始值。

```
enum Direction{
    Up,        // 0
    Down = 20, // 20
    Left,      // 21
    Right      // 22
}
const Dir: Direction = Direction.Up;
console.log(Direction.Up,Direction.Up,Direction.Left, Direction.Right)
//0 20 21 22
```

4、因为枚举成员都是`number`类型，所以我们可以直接设置为`number`

```
enum Direction{
    Up,        // 0
    Down = 20, // 20
    Left,      // 21
    Right      // 22
}
const Dir: number = Direction.Up;
console.log(Direction.Up,Direction.Up,Direction.Left, Direction.Right)
//0 20 21 22
```

5、`number`类型可以直接赋值给数值型枚举类型，必须在枚举列表中。

```
enum Direction{
    Up,        // 0
    Down = 20, // 20
    Left,      // 21
    Right      // 22
}
const Dir: Direction = 30;//报错
const Dir: Direction = 20;//正确
```

## 枚举成员映射

- 不论是哪种类型的枚举，都可以通过成员名去访问枚举成员值。

- 所以类型枚举都是可以通过枚举成员名去访问枚举成员值。

  **但是只有数值型枚举还可以通过枚举成员值去访问到枚举成员的名。**

```

   A=1，
   B=2
}
enum Str{
   C= 'xx',
   D = 'xxx'
}
console.log(Bool.A, Bool.B) // 1 2
console.ldg(Str.C,Str.D)// 'xx' 'xxx'
console.log(Bool[Bool.A]，Bool[2]) // A B
console.log(Str[Str.C]，Str['xxx']) //报错
```

## 常量枚举成员

每个枚举成员都有一个值，我们就根据成员的值定义可以将枚举成员分成常量枚举成员:初始值是

常量枚举表达式，那么该成员是常量枚举成员。

若枚举成员没有初始值，`ts`会自动给个0;

若枚举没有初始值，且上一个成员是数值的话，那该成员会是上一个成员加1，但是假如不是数值的话，就会报错。

```

    A ='a',

    B = A,

    C = 'C',

    D = -1,

    E = 1 + 2,

    F = (1+2) * 3

}
```

# 顶端类型

1、是一种通用。里，也称为记企业

2、是所有类型的父类。或者说所有类型都是顶类型的子类。

## 1、`any`

`TS1.0` `any`关键字作为标识。

允许执行任意的操作而不会产生编评错误。

2、`any`是所有类型的父类型，所以`TS`允许将`any`赋值给任意类型:

3、在程序中，如果我们使用了`any`，就代表跳过编译器的类型检查。

4、我们应该尽可能少使用`any`类型

```
let x: any;
let num:number =x
let str:string =x
let nu:null =x
x=true;
x=1;
x=10n;
x=undefined;
x=null;
```

## 2、`unknow`

1、`Ts3.0` 使用`unknown`关键字作用标识。

2、任何其它类型都可以赋值给`unknown`类型。

3、`unknown`类型只允许赋值给`unknown`类型和`any`类型，不能赋值给其它的类型，这个和`any`是不一样的，所以说它的安全性要比 `any`高。

4、不允许绝大部分的操作。

5、在使用`unknown`类型时，必须将其细化为某种具体类型。

```
let ii:unknown 
let a1: any;
ii = true; 
ii=123;
ii = undefined; 
ii = null; 
ii =Symbol( )
let x2:string = ii;//报错
let x2:string = a1;//不报错

function f1(mes: any){
  return mes.length;//不报错
}

function f2(mes: unknown){
  return mes.length//报错
}


function f2(mes: unknown){
    if(typeof mes === 'string'){
      return mes.length//报错
    }
}
```

## 总结:

1、`TS`只有`any`和`unknown`两个顶端类型

2、`TS`中的所有类型都可以赋值给`any`和`unknown`类型，

3、`any`也可以赋值给其它类型，但是`unknown`只能赋值给 `unknown`类型和`any`类型。

4、在使用`unknown`类型之前，必须细化为某种类型。

5、`unknown`相当于更加安全的`any`类型。

# 尾端类型

1、是所有其它类型的子类型。

2、也许会出现我们想要一个变量又是数值又是字符串，所以我们就需要一个尾端类型进行定义。

3、也称作0类型或者空类型，它是不包含任何值。

4、尾端类型只有一种:`never`类型

## `never`类型

1、`TS 2.0`引入 使用`never`关键字来定义。

2、`never`类型允许赋值给任何类型;

3、`never`类型是类型结构中的最底层，所以呢，任何类型都不能赋值给`never`类型。

#### 应用场景:

场景一:`never`类型可以作用函数的返回值类型，它表示该函数无法返回任何值。

```
function f():never{
    throw new Error()
}
```

场景二:死循环，永远不可能结束。

```
function infini( ):never{
    while(true){
   	 console.log('12321')
    }
}
```

场景三:使用`never`来完成场景运算;

```
type Exclude<T,U>I = T extends U ? never:T
```

场景四:类型推断，假如编译时，推断没有可用的类型，那么推断结果就是`never`类型。

```
function getLength(mes: string){
    if(typeof mes == 'string'){
       mes;
    else{
       mes; //never
    }
}
```

## 只读数组

仅允许程序读取数组元素而不允许修改数组元素。

#### 三种方式定义:

##### 1、`ReadonlyArray<数组元素类型:T>:`内置类型

`TS`早期版本提供的，定义只读数组类型。 

##### 2、`readonly`:修饰符

`TS3.4`版本引入一种新语法。使用`readonly`: 修饰符来定义只读数组。

##### 3、`Readonly<T>` : 工具类型

`TS`提供的一种内置工作类型，能够将类型参数`T`的所有属性都转换为只读

```
const red: ReadonlyArray<number> = [1,2,3] 
red[1]=4 //报错
const red1: readonly number[]=[1,2,3] 
red1[1]=3//报错
const red2: Readonly<number[]}> = [1,2,3]
```

#### 注意事项:

1、我们可以通过数组元素索引来进行访问，但是不能修改。

2、在只读数组中也不支持任何修改数组的方法。

3、在进行赋值操作中允许将常规数组赋值给只读数组类型,但是不允许反之。

```
const red: ReadonlyArray<number> = [1,2,3]
red[1] =4  //报错
red.push(1) //报错
const red5: number[]=[0];
const red6: readonly numberl]=red5;  //不报错
const red5: number[] =red6; //报错
```

# 元组类型

1、js没有的，是TS类型进行一个扩展类型。

2、和数组类型很相似。在TS中，元组类型是数组类型的子类型。元组类型长度是固定的数组类型，并且每一个元素都要有确定的类型。

3、定义:`[T0,T1,T2...]T0,T1...`针对每一个元组元素中的类型。

4、长度不匹配，类型不匹配都会报错。

```
const point: [number, string] =[1,'3']
```
