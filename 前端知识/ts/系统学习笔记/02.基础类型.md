# 二、`strictNullChecks`

`--strictNullChecks`:是否严格的检查`null`和`undefined`类型:

1、默认是`false`

2、设置为`true`之后，`undefined`和`null`只能赋值给它本身的类型

3、`undefined`和`null`是不同两种类型，不能互换使用;

# 三、`void`类型

`void`:表示某个值不存在，用作函数返回值类型。

1、假如一个函数没有返回值，那么我们就可以定义它为`void`类型。

2、使用`void`关键字去定义类型

3、当使用定义类型为`void`时，在`strictNullChecks`开启时，只能返回`undefined`,关闭

# 四、`enum`类型

`enum`:是计算机语言中的一种数据类型。枚举类型，
在`TS`中，`enum`是一种原始类型，它通过关键字`enum`来定义的。分类:根据枚举类型中的成员进行的

## 数值型枚举

1、`number`类型的子类型。它由一组命名的数值常量构成。

2、我们可以使用访问对象属性一样去访问枚举成员。

```
enum Direction{
    Up,// 0
    Down,// 1
    Left, // 2
    Right // 3
}
const Dir: Direction = Direction.Up;
console.log(Direction.Up,Direction.Up,Direction.Left, Direction.Right)
//0 1 2 3
```

3、每个数值型枚举成员都表示一个具体的数字，如果在定义时没有设置，TS会自动计算枚举成功的值，会给第一项设置为0，后面的，则是等于前一个枚举成员的值加1;可以在定义时，为一个或者多个枚举成员设置初始值。

```
enum Direction{
    Up,        // 0
    Down = 20, // 20
    Left,      // 21
    Right      // 22
}
const Dir: Direction = Direction.Up;
console.log(Direction.Up,Direction.Up,Direction.Left, Direction.Right)
//0 20 21 22
```

4、因为枚举成员都是`number`类型，所以我们可以直接设置为`number`

```
enum Direction{
    Up,        // 0
    Down = 20, // 20
    Left,      // 21
    Right      // 22
}
const Dir: number = Direction.Up;
console.log(Direction.Up,Direction.Up,Direction.Left, Direction.Right)
//0 20 21 22
```

5、`number`类型可以直接赋值给数值型枚举类型，必须在枚举列表中。

```
enum Direction{
    Up,        // 0
    Down = 20, // 20
    Left,      // 21
    Right      // 22
}
const Dir: Direction = 30;//报错
const Dir: Direction = 20;//正确
```

# 七、顶端类型

1、是一种通用。里，也称为记企业

2、是所有类型的父类。或者说所有类型都是顶类型的子类。

## 1、`any`

`TS1.0` `any`关键字作为标识。

允许执行任意的操作而不会产生编评错误。

2、`any`是所有类型的父类型，所以`TS`允许将`any`赋值给任意类型:

3、在程序中，如果我们使用了`any`，就代表跳过编译器的类型检查。

4、我们应该尽可能少使用`any`类型

```
let x: any;
let num:number =x
let str:string =x
let nu:null =x
x=true;
x=1;
x=10n;
x=undefined;
x=null;
```

## 2、`unknow`

1、`Ts3.0` 使用`unknown`关键字作用标识。

2、任何其它类型都可以赋值给`unknown`类型。

3、`unknown`类型只允许赋值给`unknown`类型和`any`类型，不能赋值给其它的类型，这个和`any`是不一样的，所以说它的安全性要比 `any`高。

4、不允许绝大部分的操作。

5、在使用`unknown`类型时，必须将其细化为某种具体类型。

```
let ii:unknown 
let a1: any;
ii = true; 
ii=123;
ii = undefined; 
ii = null; 
ii =Symbol( )
let x2:string = ii;//报错
let x2:string = a1;//不报错

function f1(mes: any){
  return mes.length;//不报错
}

function f2(mes: unknown){
  return mes.length//报错
}


function f2(mes: unknown){
    if(typeof mes === 'string'){
      return mes.length//报错
    }
}
```

## 总结:

1、`TS`只有`any`和`unknown`两个顶端类型

2、`TS`中的所有类型都可以赋值给`any`和`unknown`类型，

3、`any`也可以赋值给其它类型，但是`unknown`只能赋值给 `unknown`类型和`any`类型。

4、在使用`unknown`类型之前，必须细化为某种类型。

5、`unknown`相当于更加安全的`any`类型。

# 八、尾端类型

1、是所有其它类型的子类型。

2、也许会出现我们想要一个变量又是数值又是字符串，所以我们就需要一个尾端类型进行定义。

3、也称作0类型或者空类型，它是不包含任何值。

4、尾端类型只有一种:`never`类型

## `never`类型

1、`TS 2.0`引入 使用`never`关键字来定义。

2、`never`类型允许赋值给任何类型;

3、`never`类型是类型结构中的最底层，所以呢，任何类型都不能赋值给`never`类型。

#### 应用场景:

场景一:`never`类型可以作用函数的返回值类型，它表示该函数无法返回任何值。

```
function f():never{
    throw new Error()
}
```

场景二:死循环，永远不可能结束。

```
function infini( ):never{
    while(true){
   	 console.log('12321')
    }
}
```

场景三:使用`never`来完成场景运算;

```
type Exclude<T,U>I = T extends U ? never:T
```

场景四:类型推断，假如编译时，推断没有可用的类型，那么推断结果就是`never`类型。

```
function getLength(mes: string){
    if(typeof mes == 'string'){
       mes;
    else{
       mes; //never
    }
}
```

## 只读数组

仅允许程序读取数组元素而不允许修改数组元素。

#### 三种方式定义:

##### 1、`ReadonlyArray<数组元素类型:T>:`内置类型

`TS`早期版本提供的，定义只读数组类型。 

##### 2、`readonly`:修饰符

`TS3.4`版本引入一种新语法。使用`readonly`: 修饰符来定义只读数组。

##### 3、`Readonly<T>` : 工具类型

`TS`提供的一种内置工作类型，能够将类型参数`T`的所有属性都转换为只读

```
const red: ReadonlyArray<number> = [1,2,3] 
red[1]=4 //报错
const red1: readonly number[]=[1,2,3] 
red1[1]=3//报错
const red2: Readonly<number[]}> = [1,2,3]
```

#### 注意事项:

1、我们可以通过数组元素索引来进行访问，但是不能修改。

2、在只读数组中也不支持任何修改数组的方法。

3、在进行赋值操作中允许将常规数组赋值给只读数组类型,但是不允许反之。

```
const red: ReadonlyArray<number> = [1,2,3]
red[1] =4  //报错
red.push(1) //报错
const red5: number[]=[0];
const red6: readonly numberl]=red5;  //不报错
const red5: number[] =red6; //报错
```

## 多余属性

对象的多余属性就可以理解为多出来的属性。多余属性是一个相对的概念，只有在比较两个对象类型的关系时说多余属性才有意义。

比如源对象类型和目标对象类型？

```
let point:{x:number, y:number} = {x:1,y:4}
```

所以我们把"="左侧叫做目标对象类型，右侧叫做源对象类型。

所以，当源对象类型中存在一个或多个在目标对象类型中不存在的属性时，我们就说源对象类型和目标对象类型存在多余属性。

```
let point:{x:number, y:number} = {x:1,y:4}
//不存在多余属性
let point:{x:number, y:number} = {x:1,y:4,z:20}
//存在多余属性z
```

#### 编译器对多余属性的检查

会影响类型见的子类型的兼容性以及赋值兼容性，也就是说编译器不允许在一些的操作中存在多余属性。

##### 体现方面：

1、直接赋值

2、作为函数参数来调用时

```
//直接赋值
let ponit:{
    x:number,
    y:number
} = {x:1,y:2,z:0}

//作用函数参数来调用时
function fn(point:{x:number,y:number}){
	console.log(point.x)
}
fn({x:1,y:2,z:0})
```

##### 为什么会存在？

1、误操作

2、在编译阶段能够带来的最直接的帮助发现属性名的拼写错误，并且还根据算法来推测可能的属性名。

```
const task:{cancel:boolean} = {canceln:true}
```

#### 如何允许多于属性

1、类型断言

- 处于类型断言表达式，编译器不再对其进行多于属性检查

```
const p1:{x:number} = {x:0,y:0} as {x:number}
```

2、使用'//@ts-ignore'注释指令

```
//@ts-ignore
const p1:{x:number} = {x:0,y:0}
```

3、添加索引签名

```
const p1:{x:number,[prop:string]:number} = {x:0,y:0,z:4}
```

4、赋值

```
const temp = {x:0,y:0}
const p3:{x:number} = temp
```

# 重载函数

是指一个函数同时拥有多个同类的函数签名。当使用不同数量和把类型的参数调用重载函数时，可以执行不同的函数实现代码

**注意：多个且同类 ===>调用签名两个及以上或者构造签名两个及以上。**

重载函数的定义由两部分组成：

- 一条或者多条函数重载语句

- 一条函数实现语句

```
//函数实现语句
function add (x:number|any[],y:number|any[]):any{
   if(typeof x === 'number' && typeof y === 'number' ){
     return x + y
   }
   if(Array.isArray(x) && Array.isArray(y) ){
     return [...x,...y]
   }
}

//函数重载语句
const a:number = add(1,2)
const b:any [] = add([1],[2])
```

## 函数重载

1、不带有函数体的声明语句叫做函数重载；不包含函数体，它只提供函数的类型信息。

```
//函数重载
function add(x:number,y:number):number
function add(x:any[],y:any[]):any[]
```

2、函数重载只在编译阶段，在编译生成`js`后会被删除，因此在最终生成的`js`代码是不包含函数重载的代码。

3、函数重载允许一个或者多个，但是只有多个时才有意义。因此只有一个时，就直接定义就好了。

4、函数重载应该位于函数实现之前，并且每一个函数重载的函数名和函数实现中的函数名必须一致。

5、函数重载与函数实现语句之间不能存在其他的语句，否则就会产生编译错误。

```
//函数重载
function add(x:number,y:number):number
function add(x:any[],y:any[]):any[]
console.log(222) //编译错误
//函数实现
function add (x:number|any[],y:number|any[]):any{
   if(typeof x === 'number' && typeof y === 'number' ){
     return x + y
   }
   if(Array.isArray(x) && Array.isArray(y) ){
     return [...x,...y]
   }
}

```

## 函数实现

1、函数实现包含了实际函数体代码，并且在编译时存在，并且在编译后生成的`js`中也同样存在。每一个重载函数只允许有一个函数实现，它只能位于所有函数重载语句之后，否则会产生编译错误。

2、我们在函数重载中的函数签名是每一个函数重载中独立存在的，也就是说每个函数重载都有一个调用签名，但是函数实现中的函数签名不属于重载函数的调用签名之一的，所以不可以混用。

 3、函数实现需要兼容每个函数重载中的函数签名的。

4、需要开发者自行去判断类型做兼容

```
//函数重载
function add(x:number,y:number):number
function add(x:any[],y:any[]):any[]

//函数实现
function add (x:number|any[],y:number|any[]):any{
   if(typeof x === 'number' && typeof y === 'number' ){
     return x + y
   }
   if(Array.isArray(x) && Array.isArray(y) ){
     return [...x,...y]
   }
}
add(1,2) //正确
add([1],[2])//正确
add([1],2)//错误
```

## 函数重载的执行顺序

当程序中调用了一个重载函数时，编译器首先构建出一个候选函数重载列表。

一个函数重载只有满足以下条件才能成为本次函数调用的候选哈纳湖重载

1、函数实际参数的数量不少于函数重载中定义的必选参数的数量

2、函数实际参数的数量不多于函数重载中定义的必选参数的数量

3、每个实际参数的类型能够赋值给函数重载定义中对应形式参数的类型

**总结：必选参数数量相等，类型相等**

```
function f(x:string):void; //重载1
function f(y:'specialized'):void; //重载2

function f(x:string){
  console.log('123')
}//函数实现

//候选列表
//function f(y:'specialized'):void; //重载2 ===>字面量类型，所以优先级要高于重载1
//function f(x:string):void; //重载1 


```

#### 解析顺序：

1、字面量类型的优先级要比普通的高

2、当我们实际调用的参数没有在候选函数列表中时，会产生编译错误

注意：因为ts没有很只能，它只能把候选函数重载列表的首位当作它最终使用的一个函数重载，所以我门需要注意函数重载时的顺序问题

```
function f(x:any):number; //重载1
function f(x:string):0|1; //重载2

function f(x:any):any{
  console.log(x)
}//函数实现

const a:0|1 = f('hi') //编译错误
这里是因为，函数重载是根据函数参数判断执行那个的，上面两个函数重载的参数都满足时('hi'既满足any又满足string)，函数重载的顺序就显得尤为重要了，这个时候会选择函数重载列表的首位当作它最终使用的一个函数重载，这个例子中，由于函数重载列表的首位为重载1，所以返回值0|1和number不对应,报错

function f(x:'string'):0|1; //重载2
function f(x:any):number; //重载1


function f(x:any):any{
  console.log(x)
}//函数实现

const a:0|1 = f('hi') //编译正确
const b = f('hi') //编译正确

```

## 重载函数的类型

函数实现的函数签名不属于重载函数的调用签名之一

调用签名的书写顺序是有意义的，它决定了函数重载的解析顺序

```
function f(x:string):void; //重载1
function f(x:number):number; //重载2

function f(x:any):any{
  console.log(x)
}//函数实现

相当于 对象类型字面量
{
  (x:string):void
  (x:number):number
}
```

## 函数中的`this`的类型

`this`是`js`中的关键字，它可以表示调用函数的对象或者实例对象等

在默认情况下，`this`的类型是`any`，并允许程序在`this`值上执行任意的操作。

因为编译器不会对`any`类型进行类型检查

## `this`类型设置

ts中是允许this作为函数的参数来定义它的类型

#### `this`参数

1、`this`必须在参数的首位

2、`this`参数类型只在编译阶段，不会再编译生成的`js`中存在

3、`this`类型为`void`时，在函数体中就不允许读写`this`的属性和方法

4、在定义了`this`的类型之后，我们就不能直接去执行，要用`Function.prototype.call`方法去执行，这种方法它能够指定`this`的值。

```
function foo(this: {name:string}):number{
   this.name = 'aa'
   return 1
}
foo()//编译错误
foo.call({name:'xxx'})//编译正确
```

# 接口类型

## 接口声明

它能够去表示任意的对象类型，不同的是既可以指定类型参数又可以给对象类型命名。

语法：

```
interface interfaceName{

   TypeMember,

   TypeMember

   ...

}
```

```
interface IMap{
   localtionName: string,
   locationY: number
}
```

#### 接口类型的类型成员分类：

- 属性签名

- 调用签名

  - 定义了该对象类型表示的函数在调用时的类型参数，参数列表以及返回值类型。

  - 语法：

    ```
    {
    
       (ParameterList)：Type
    
    }
    
    ParameterList：参数列表
    
    Type：函数返回值类型
    ```

    ```
    interface IMap{
       (message?:string,num?:number):Error
    }
    ```

- 构造签名

  - 定义了该对象类型表示的构造函数在使用`new`运算符调用时的参数列表以及返回值类型

  - 语法：

    ```
    {
    
      new (ParameterList)：Type
    
    }
    
    ParameterList：参数列表
    
    Type：构造函数返回值类型
    ```

    ```
    interface IMap{
       new (message?:string,num?:number):Error
    }
    ```

- 方法签名

  - 是声明函数类型的属性成员的简写

  - 语法：

    ```
    PropertyName(ParamList):Type
    PropertyName:方法名称，可以为标签符、字符串、数字和可计算属性名
    ParamList：表示可选方法形式参数列表类型
    Type：可选的方法返回值类型
    
    相当于在调用签名前面加了一个方法名称PropertyName	
    ```

    ```
    interface IMap{
       getElementById (elementId:string):HTMLElement | null
    }
    ```

    

  - 之所以说它是声明函数类型的属性成员的简写，是因为方法签名可以改写为语法稍微复杂的属性签名。

    ```
    PropertyName:{(ParamList):Type}
    ```

    ```
    interface IMap{
      f(x:string):string //方法签名
      f:{(x:string):string} //属性签名+对象类型字面量
    }
    ```

    

  - 在改写的语法中，属性名保持不变并使用对象类型字面量和调用签名来表示函数类型，因为该对象类型字面量中只有一个调用签名，因此可以再次使用函数类型字面量来代替对象类型字面量

    ```
    PropertyName:(ParamList) => Type
    ```

    ```
    interface IMap{
      f(x:string):string//方法签名
      f:(x:string)=>string//属性签名+函数类型字面量
    }
    ```

- 若接口中包含多个名字相同但参数列表不同的方法签名成员，则表示该方法是重载方法，它具有多种调用签名。

  ```
  interface IMap{
    f(x:string):string
    f():number
    f(x:boolean):boolean
  }
  ```

  

- 索引签名

  - `js`支持使用索引去访问对象的属性的，即通过"[]"语法去访问对象的属性。

    ```
    const colors = ['red','green','blue']
    const red = colors[0]
    ```

  - 接口中的索引签名能够描述使用索引访问的对象属性类型

  - 字符串索引签名

    ```
    [indexName:string]:Type
    indexName:索引名称，它可以为任意合法的标识符，索引名只是占位，它不代表真实索引名称。在字符串索引签名中，索引名的类型必须是string类型，Type就代表着索引值的类型。
    ```

    

    ```
    interface A{
     [prop:string]:number; //只能是键名为string，键值为number
     a:number,
     b:0,
     c:1|2，
     d:'555' //报错
    }
    ```

    

  - 数值索引签名
