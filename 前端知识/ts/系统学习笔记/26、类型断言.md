# 类型断言

我们开发者使用类型断言来告诉编译器某个表达式的具体类型。

## 类型断言语法

```
<T>expr
```

`T`表示类型断言的目标类型，`expr`表示一个表达式，`<T>`类型断言尝试将表达式`expr`的类型转换成T类型。

```
<HTMLInputElement>username
```

运算符优先级问题：`username.value`会把这整体当作一个表达式，要解决这个问题，可以加括号

```
(<HTMLInputElement>username).value
```

## 类型断言-`as`语法

语法：

```
expr as T
```

起初`TS`只支持`<T>`，但是`React ===> jsx`，它的语法和`<T>`会冲突，所以添加了`as T`语法，并且在`JSX`中只支持`as T`这种语法。

## 类型断言的约束

类型断言不允许在两个类型之间随意做转换的，需要一些前提条件：

1、`expr`表达式的类型能够赋值给`T`类型

2、`T`类型能够赋值给`expr`表达式的类型

例如：`HTMLElement ===> HTMLInputElement` 

这个条件意味着，在执行类型断言时编译器会尝试进行双向的类型兼容性判定，允许将一个类型转换为更精确的类型或者更加宽泛的类型。

```
interface point2d{
  x:number;
  y:number;
}
interface point3d{
  x:number;
  y:number;
  z:number;
}
const p2d:point2d = {x:0,y:1}
const p3d:point3d = {x:0,y:1,z:2}

//point2d 转 point3d 广泛的范围
const p0 = p2d as point3d
p0.x;
p0.y;
p0.z;

//point3d 转 point2d 更精确的
const p1 = p3d as point2d
p1.x;
p1.y;

let a:boolean = 'aaa' as boolean; //报错
let b:boolean = 'aaa' as unknow as boolean;//正确
```

## `const`类型断言

`const`类型断言是一种特殊形式的`<T>`类型断言和`as T`类型断言，它能够将某一类型转换成不可变类型。

语法形式：

```
  expr as const
  <const> expr
```

```
let a1 = true
let a2 = true as const
a1 = false
a2 = false // 报错
a2 = true

let b1 = 'hello'
let b2 = <const>'ccc'
b1 = '123'
b2 = 'xxx'//报错
b2 = 'ccc'

let a1 = {x:0,y:1}
let b1 = {x:0,y:1} as const
===>
let b1:{
  readonly x:0;
  readonly x:1;
}
```
