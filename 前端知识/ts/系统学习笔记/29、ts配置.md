# `TS`配置

## 安装并使用`TS`编译器来编译`TS`程序

node 环境

全局安装TS语言：

```
npm i -g typescript
tsc -v 版本号
```

简单基本语句使用：

```
tsc --help 帮助
tsc -v 版本号
tsc --init 初始化tsconfig.json
```

## 编译

#### 编译单个文件

```
tsc index.ts
```

#### 编译多个文件

```
tsc index.ts utils.ts demo.ts ...
```

通配符来模块匹配待编译的文件：

`*`：匹配零个或多个字符，但不包含目录分隔符

`?`：匹配一个字符，但不包含目录分隔符

`**/`：匹配任意目录及其子目录

```
tsc *.ts
```

#### 观察模式

编译器会监视文件的修改并自动重新编译文件。通过--watch（-w）编译选项开启

## `tsconfig.json`

#### `tsconfig.json`中的`include`

使用顶层属性`include`属性能够定义编译文件列表，也可以使用通配符

```
//tsconfig.json
{
  "include":["bar/*.ts"]
}
```

#### `tsconfig.json`中的`exclude`

使用顶层属性`exclude`在匹配到的文件列表中去除指定文件，也支持通配符的使用。

必须和`include`一起使用

```
//tsconfig.json
{
  "include":["**/*.ts"],
  “exclude":["**/b.ts"]
}
```

## 编译`JS`

如何配置`TS`让它去编译`JS`

`allowJs` 启用编译选项后，指定编译路径：

```
tsc demo.js --allowJs --outDir dist
```

才可以编译`js`文件

## 如何对`JS`进行类型检查

`--checkJS`

```
tsc demo.js --allowJs --checkJS --outDir dist
```
