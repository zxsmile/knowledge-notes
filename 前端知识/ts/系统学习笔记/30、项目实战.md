# 项目实战

## 1、`TS`与`Babel`

`Babel`能够对输入的`JS`代码进行解析，并在保证功能不变的情况下重新生成`JS`代码。

结合`TS`：`TS` ===>`TS`静态类型检查 ===>`Babel`转译`JS`===>`JS`文件

`babel/core` 提供`Babel`核心的转译代码功能

`babel/cli` 提供`babel`命令行工具

`babel/perset-env` 提供自动检测待转译的语法以及自动导入填充脚本

`babel/preset-typescript` 它是`babel`和`TS`结合的关键，它提供从`TS`代码中删除类型相关代码的功能。如类型注解。

#### `babel`配置

```
//.babelrc.json
{
   "presets":[
      "@babel/env",
      "@babel/typescript"
   ]
}
```

#### 配置静态类型检查

`--noEmit`编译选项：编译器只会对代码进行静态类型检查而不会生成`JS`文件

```
tsc --noEmit
```

#### 注意事项

有一些TS代码Babel处理不好，例如：

- 命名空间

- `<T>`类型断言语法 ===> x as T支持

- 常量枚举"`const enum`"

- 跨文件声明的枚举，在TS中会被合并成一个声明

  使用建议：现在有的项目使用babel，花式使用Babel进行转译 + 结合TS静态类型检查，新的项目，还是建议直接使用TS来进行转译

## 2、`webpack`

#### 使用打包工具的原因

1、由于运行环境（浏览器）不支持TS代码中使用的模块格式（ES模块）因此会导致无法加载代码。打包工具能够解析模块间的依赖关系并将多个模块文件合并为运行环境能够加载的单一文件。

2、在浏览器环境中减少加载的资源文件数量能够提供web应用的性能。

#### `webpack`主要用途

把多个`JS`打包成一个或者多个`JS`文件

`ts + webpack`:

把多个`TS`源文件打包成一个单独的`JS`文件

## 3、`ESLint`

#### 静态程序分析工具：

在代码运行之前对代码进行分析，能够发现代码中存在的缺陷，并有助于提高代码的质量。

#### `ESLint`

一种`JS`语言的静态程序分析工具

一句预定义的一组规则来检查代码，当发现代码中没有按规则来，就给出相应提示。

`vscode`内置对TS语言的支持

代码格式化
