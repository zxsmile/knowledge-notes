# 一、元组类型

1、`js`没有的，是`TS`类型进行一个扩展类型。

2、和数组类型很相似。在TS中，元组类型是数组类型的子类型。元组类型长度是固定的数组类型，并且每一个元素都要有确定的类型。

3、定义:`[T0,T1,T2...]T0,T1...`针对每一个元组元素中的类型。

4、长度不匹配，类型不匹配都会报错。

```
const point: [number, string] =[1,'3']
```

## 只读元组:

- `readonly`修饰符
- `ReadOnly<T>` 工具类

```
const.rPonit: readonly [number, string] = [1,'xxx'] 
rPonit[1]='xxxxx'//报错
const nPoint: Readonly<[number, string]> = [1,'xxxxx']
nPoint[0]= 123 //报错
```

**在给只读元组赋值时，允许将常规元组赋值给只读元组类型,但是不允许反之。**
