# 一、数组类型

**注意:这两种表示法编码风格有区别，功能上没有任何区别**

## 简便数组类型表示法:

- 数组元素类型之后添加"[]"
- 复合类型表示方式:在元素类型上使用分组运算符，也就是"()"

```
const digi: number[] =[1,2,3]
const red: string[] =['1', '2'1
const user:(string | number | boolean)[]=['zhangsan',18,true]
```

## 泛型数组类型表示法:

```
const digits: Array<number> = [1,2,34,5]	
const userInfo: Array<number | string> =  [1, 2,3,5, '123']
```


使用泛型来表示：`Array<类型参数|类型参数>`

## 两种方法比较:

1、在定义简单数组类型时，比如数组元素为单一的原始类型就可以直接使用简便类型更加清河和简洁。

2、比如复杂的类型，有两种或者多种时，就可以使用泛型数组表示法。

## 数组元素类型

在定义了数组类型之后，当访问数组元素时，可以得到正确的数据类型。

**注意:当访问数组中不存在的元素时，将返回`undefined`,但是 `TS`类型系统中无法推断你是否存在越界的问题，所以还是会给返回当前你定义的数组元素的类型**

```
let arr: number[] = [1,2, 3, 4,5]
const zero = arr[0] 
const zero1 = arr[100] //越界了，类型依然是number
```

## 只读数组

仅允许程序读取数组元素而不允许修改数组元素。

#### 三种方式定义:

##### 1、`ReadonlyArray<数组元素类型:T>:`内置类型

`TS`早期版本提供的，定义只读数组类型。 

##### 2、`readonly`:修饰符

`TS3.4`版本引入一种新语法。使用`readonly`: 修饰符来定义只读数组。

##### 3、`Readonly<T>` : 工具类型

`TS`提供的一种内置工作类型，能够将类型参数`T`的所有属性都转换为只读

```
const red: ReadonlyArray<number> = [1,2,3] 
red[1]=4 //报错
const red1: readonly number[]=[1,2,3] 
red1[1]=3//报错
const red2: Readonly<number[]}> = [1,2,3]
```

#### 注意事项:

1、我们可以通过数组元素索引来进行访问，但是不能修改。

2、在只读数组中也不支持任何修改数组的方法。

3、在进行赋值操作中允许将常规数组赋值给只读数组类型,但是不允许反之。

```
const red: ReadonlyArray<number> = [1,2,3]
red[1] =4  //报错
red.push(1) //报错
const red5: number[]=[0];
const red6: readonly numberl]=red5;  //不报错
const red5: number[] =red6; //报错
```
