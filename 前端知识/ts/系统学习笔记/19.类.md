# 一、类的概念

在ES6时，JS引入类的概念，但其本质还是函数，只是一种语法糖。

TS对JS中的类进行了扩展，为其添加了类型支持，如实现接口，泛型等。

JS中class关键字，TS定义类也是class关键字。类似于函数定义，类的定义也有两种方式：

- 类声明
- 类表达式

# 二、类声明

```
class ClassName{
  //...
}
class 关键字
ClassName 是类名称（必写的，一般首字母大写）
```

#### 特点：

1、函数声明是提升，但是类不会被提升，必须先声明后使用。

2、在使用类声明时，不允许声明同名的类，会产生错误。

```
class Circle{

}
const a = new Circle()
```

# 三、类表达式

```
const Name = class ClassName{
  //...
}
Name 引用类的变量名
class 关键字
ClassName 是类名称（可选）
```

省略ClassName时，就叫做匿名类

```
const Circle = class {

}
const a = new Circle()
```

如果在类表达式中定义了类名，类名只能在类内部使用，不可以在类外引用。

```
const Circle = class A{
  name = A.name
}
const a = new A()//报错
const a = new Circle()//正确
```

# 六、`readonly`

在成员变量前添加`readonly`修饰符，可以表示该成员变量是只读的，必须在声明时初始化或在构造函数中初始化。

```
class A{
  readonly a:number = 1 //初始化
  readonly b:number
  constructor() {
     this.b = 1 //初始化
  }
  m() {
    this.a = 3  //报错
  }
}
```

# 八、成员存取属性

成员存取也是`get`和`set`方法构成，且会在类中声明一个属性：

1、`get`和`set`的属性类型是一致的

```
class A{
  a:number = 1
  get():number{
    return this.a
  }
  set(value:number){
    this.a = value
  }
}
```

2、`get`方法和`set`方法必须具有相同的可访问性

```
// 正确
class A{
  private a:number = 1
  private get():number{
    return this.a
  }
  private set(value:number){
    this.a = value
  }
}

//报错
class A{
  private a:number = 1
  private get():number{
    return this.a
  }
  public set(value:number){ 
    this.a = value
  }
}

// 正确
class A{
  private a:number = 1
  public get():number{
    return this.a
  }
  public set(value:number){
    this.a = value
  }
}
```

3、可以在存取器方法中写一些逻辑来控制

```
class A{
  a:number = 1
  get():number{
    return this.a
  }
  set(value:number){
    if(value > 0){
      this.a = value
    }
  }
}
```

# 九、成员的可访问性

定义类的成员允许在何处被访问。`TS`为类成员提供了三种可访问性修饰符：

- `public`
- `private`
- `protected`

这三种可访问性修饰符，是`TS`语言对JS语言的补充，在JS语言中不支持这三种可访问修饰符。

#### `public`

1、类的公有成员没有访问限制，也就是说在内部、外部、以及派生类有内部访问

2、默认情况下，类的所有成员都是公有成员，所以省略`public`这个修饰符

```
class Base{
  public a:string = '1'
}
 class Device extends Base{
      public b() {
         return this.a
      }
 }
 
 let deviced = new Device()
 console.log(deviced.a,deviced.b)
```

#### `protected`

允许在类的内部以及派生类的内部访问

```
class Base{
  protected a:string = '1'
  b() {
         return this.a
      }
}
 class Device extends Base{
      b() {
         return this.a
      }
 }
 
 let deviced = new Base()
 console.log(deviced.a) //错误
 
 let deviced = new Device()
 console.log(deviced.a)//错误
```

##### `private`

类的私有成员，只允许在当前类的内部被访问，在当前类的外部还是派生类都不允许被访问。

```
class Base{
  private a:string = '1'
  b() {
         return this.a
      }
}
 class Device extends Base{
      b() {
         return this.a //报错
      }
 }
 
  let deviced = new Base()
 console.log(deviced.a) //错误
```

# 十一、构造函数

用于创建和初始化类的实例。当使用`new`运算符调用一个类时，类的构造函数就会被调用，构造函数以`constructor`作用函数名。

1、看可以定义可选参数、默认值参数、剩余参数。

```
class Circle{
   radius:number;
   constructor(r:number = 0, b?:string, ...c:string[]){
      this.radius = r
   }
}
const c = new Circle(1)
```

2、不允许定义返回值类型。

3、可以使用修饰符，但是假如你设置了`private`只能在当前类中使用。

```
class Circle{
   radius:number;
   private constructor(r:number = 0, b?:string, ...c:string[]){
      this.radius = r
   }
}
const c = new Circle(1)//编译错误
```

4、支持函数重载，可0个或者多个，但是函数实现只能一个。

```
 class Circle{
   radius:number;
   constructor(x:number, y:number) ;
   constructor(s:string);
   constructor(xs:number | string, y?:number){
   
   }
}
const c = new Circle(1)
```

# 十二、参数成员

在`TS`中，提供了一种简介语法，能够把构造函数的形式参数声明为类的成员变量，它叫作参数成员。

也就是说当我们给形参添加可访问性修饰符或者`readonly`修饰符时，这个形参就变成了类的成员变量。

注意：`readonly`可以和我们的可访问性修饰符进行叠加使用。

```
class A{
  constructor{
     public x:number,
     protected readonly y:number,
     private readonly z:number
  }
}
```

# 十四、重写基类成员

1、也就是说可以在子类去重写父类的成员变量和成员函数的

2、在子类与父类中同名的成员变量及函数在实现继承中会被重写，也就是替换。

```
class A{
  name:string = '';
  age:number = 1;
  getInfo() {
    return this.age
  }
}

class B extends A{
  name:string = '111';
  age:number = 10;
}

const b = new B()
console.log(b.name)//'111'
console.log(b.getInfo)
```

3、当存在同名的非私有成员时，要在子类中访问父类，可以通过`super`关键字。

4、当在父类中写了访问性时，在子类中同名时只能往大扩展不能缩小范围，不能从`public`变成`private`

```
class A{
  protected name:string = '';
  protected age:number = 1;
  getInfo() {
    return this.age
  }
}

class B extends A{
  public name:string = '111';
  protected age:number = 10;
}

const b = new B()
console.log(b.name)//'111'
console.log(b.getInfo)
```

5、子类重写父类时，类型要一直，不能改变类型

```
class A{
  protected name:string = '';
  protected age:number = 1;
  getInfo() {
    return this.age
  }
}

class B extends A{
  public name:number = 22;//编译错误
  protected age:number = 10;
}
```

# 十五、派生类实例化

1、在派生类的构造函数中必须调用基类的构造函数，否则是不能正确的实例化派生类的

2、调用方法就是'`super`'调用：调用基类的构造函数

3、且要在使用`this`前去调用

#### 初始化顺序：

1、初始化基类的属性

2、调用基类的构造函数

3、初始化派生类的属性

4、调用派生类的构造函数

```
class A{
  name:string = 'black';//1
  constructor() {
     this.color = 'black';//2
  }
}

class B extends A{
  radius:number;   //3
  constructor() {  //4
     super()
     this.color = 'white'
  }
}

const b = new B()
b.radius;
b.color;
```

# 十八、静态成员

类的定义中可以包含静态成员，类的静态成员只属于类本身，只允许通过类名来访问。

`static`关键字进行定义

```
class A{
  static version:string = '1.0'
}
const version = A.version
const c = new A()
```

# 十九、静态成员的可访问性

类的静态成员也可以定义不同的可访问性：`public`、`private`、`protected`

类的`public`静态成员对访问没有限制，可以在当前类的内部、外部以及派生类的内部访问。

```
class B{
  public static x:string = ''
  a() {
    B.x
  }
}

class C extends B{
  b() {
     B.x
  }
}

C.x
B.x
```

类的`protected`静态成员允许在当前类以及派生类内部进行访问

```
class B{
  protected static x:string = ''
  a() {
    B.x
  }
}

class C extends B{
  b() {
     B.x
  }
}

C.x //报错
B.x//报错
```

类的`private`静态成员只允许在当前类的内部进行访问

```
class B{
  private static x:string = ''
  a() {
    B.x
  }
}

class C extends B{
  b() {
     B.x //报错
  }
}

C.x //报错
B.x //报错
```

# 二十、`this`类型

在类中存在一种特殊的`this`类型，代表的是当前this值的类型，我们可以在类的非静态成员的类型注解中使用`this`类型

```
class Counter{
  private count:number = 0;
  public add():this{
    this.count++
    return this
  }
  public sub():this{
    this.count--
    return this
  }
  public get():number{
    return this.count
  }
}

const counter = new Counter
counter.add().sub().get()
```

1、`this`是动态的

2、`this`不允许应用于类的静态成员

```
class A{
  static a: this; // 编译错误
  foo():this{
    return this
  }
}

class B extends A{
  bar():this{
    return this
  }
}

const b = new B();
const x = b.bar().foo()
```

# 二十一、静态成员的继承

在包含静态成员的类中，可访问性是`public`静态成员或者是`protected`静态成员或者不写，可以被继承，`private`私有的不会被继承。

```
class Base{
  public static x:string = '';
  protected static y:string = '';
  private static z:string = '';
}

class B extends Base{
  b(){
     B.x;
     B.y;
     B.z; //报错
  }
}
```
