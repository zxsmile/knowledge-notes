# 一、类的概念

在ES6时，JS引入类的概念，但其本质还是函数，只是一种语法糖。

TS对JS中的类进行了扩展，为其添加了类型支持，如实现接口，泛型等。

JS中class关键字，TS定义类也是class关键字。类似于函数定义，类的定义也有两种方式：

- 类声明
- 类表达式

# 二、类声明

```
class ClassName{
  //...
}
class 关键字
ClassName 是类名称（必写的，一般首字母大写）
```

#### 特点：

1、函数声明是提升，但是类不会被提升，必须先声明后使用。

2、在使用类声明时，不允许声明同名的类，会产生错误。

```
class Circle{

}
const a = new Circle()
```

# 三、类表达式

```
const Name = class ClassName{
  //...
}
Name 引用类的变量名
class 关键字
ClassName 是类名称（可选）
```

省略ClassName时，就叫做匿名类

```
const Circle = class {

}
const a = new Circle()
```

如果在类表达式中定义了类名，类名只能在类内部使用，不可以在类外引用。

```
const Circle = class A{
  name = A.name
}
const a = new A()//报错
const a = new Circle()//正确
```

# 四、成员变量

```
class Circle{
  num:number = 1 //初始化
}
```

```
class Circle{
  num:number
  constructor() {
     this.num = 1 //初始化
  }
}
```

# 五、`strictPropertyInitialization`

`strictPropertyInitialization`：帮助严格检查未经初始化的成员变量

# 六、`readonly`

在成员变量前添加`readonly`修饰符，可以表示该成员变量是只读的，必须在声明时初始化或在构造函数中初始化。

```
class A{
  readonly a:number = 1 //初始化
  readonly b:number
  constructor() {
     this.b = 1 //初始化
  }
  m() {
    this.a = 3  //报错
  }
}
```

# 八、成员存取属性

成员存取也是`get`和`set`方法构成，且会在类中声明一个属性：

1、`get`和`set`的属性类型是一致的

```
class A{
  a:number = 1
  get():number{
    return this.a
  }
  set(value:number){
    this.a = value
  }
}
```

2、`get`方法和`set`方法必须具有相同的可访问性

```
// 正确
class A{
  private a:number = 1
  private get():number{
    return this.a
  }
  private set(value:number){
    this.a = value
  }
}

//报错
class A{
  private a:number = 1
  private get():number{
    return this.a
  }
  public set(value:number){ 
    this.a = value
  }
}

// 正确
class A{
  private a:number = 1
  public get():number{
    return this.a
  }
  public set(value:number){
    this.a = value
  }
}
```

3、可以在存取器方法中写一些逻辑来控制

```
class A{
  a:number = 1
  get():number{
    return this.a
  }
  set(value:number){
    if(value > 0){
      this.a = value
    }
  }
}
```

# 九、成员的可访问性

定义类的成员允许在何处被访问。`TS`为类成员提供了三种可访问性修饰符：

- `public`
- `private`
- `protected`

这三种可访问性修饰符，是`TS`语言对JS语言的补充，在JS语言中不支持这三种可访问修饰符。

#### `public`

1、类的公有成员没有访问限制，也就是说在内部、外部、以及派生类有内部访问

2、默认情况下，类的所有成员都是公有成员，所以省略`public`这个修饰符

```
class Base{
  public a:string = '1'
}
 class Device extends Base{
      public b() {
         return this.a
      }
 }
 
 let deviced = new Device()
 console.log(deviced.a,deviced.b)
```

#### `protected`

允许在类的内部以及派生类的内部访问

```
class Base{
  protected a:string = '1'
  b() {
         return this.a
      }
}
 class Device extends Base{
      b() {
         return this.a
      }
 }
 
 let deviced = new Base()
 console.log(deviced.a) //错误
 
 let deviced = new Device()
 console.log(deviced.a)//错误
```

##### `private`

类的私有成员，只允许在当前类的内部被访问，在当前类的外部还是派生类都不允许被访问。

```
class Base{
  private a:string = '1'
  b() {
         return this.a
      }
}
 class Device extends Base{
      b() {
         return this.a //报错
      }
 }
 
  let deviced = new Base()
 console.log(deviced.a) //错误
```
