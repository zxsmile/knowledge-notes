# 1、性能指标

#### `FCP`（`First Contentful Paint`）⾸次内容绘制

##### 什么是`FCP`？

`FCP` ，⾸次内容绘制。该指标⽤于测量从⽹⻚加载到任何⼀部分呈现在屏幕上的时间。⽤⼾访问

`Web` ⻚⾯过程中，在 `FCP` 之前⻚⾯处于⽩屏，没有实际内容。（如下图所⽰）

![项目1](.\images\项目1.png)

为了达到良好的⽤⼾体验，⽹⻚应尽⼒将 `FCP` 控制在1.8秒以内，不妨将⽹⻚加载的第75百分位作为衡量阈值。

![项目2](.\images\项目2.png)

- ⼩于 1.8 秒，表⽰⻚⾯加载较快，⽩屏时间短
- 1.8 秒⾄ 3 秒，表⽰⻚⾯加载较慢，但基本能接受
- 3 秒以上，表⽰⻚⾯加载很慢，⽩屏时间过⻓

#### `LCP`（`Largest Contentful Paint`）最⼤内容绘制

`LCP` ，最⼤内容绘制。该指标⽤来测量从⽹⻚加载到⽹⻚最⼤内容（⽂本块/图⽚元素）呈现在屏幕上的时间。

![项目3](.\images\项目3.png)

最⼤内容元素有哪些？

- `<img>` 元素
- `<svg>` 内的 `<image>` 元素

- 包含海报图⽚的` <video> `元素
- 为⾃动播放` <video> `元素⽽绘制的第⼀帧
- 动画图⽚（如 `GIF `动画）的第⼀帧
- 包含⽂本节点或其他内嵌级别⽂本元素的⼦项的块级元素

为了达到良好的⽤⼾体验，⽹⻚应尽⼒将 `LCP` 控制在2.5秒以内，不妨将⽹⻚加载的第75百分位作为衡量阈值。

![项目4](.\images\项目4.png)

#### `FID`（`First Input Delay`）

`FID` ，测量的是⽤⼾⾸次与⽹⻚互动（点链接、点按钮等）到浏览器响应该互动的时间。

#### `TTFB` ⾸字节时间

`TTFB` ，⽤于测量资源请求与响应第⼀个字节之间的时间。

![项目5](.\images\项目5.png)

#### `TTI`（`Time To Interacive`）可交互时间

`TTI`，⽤于测量从⽹⻚开始加载到主要⼦资源已加载且能够快速响应⽤⼾输⼊的时间。（`TTI`，即是静默期之前的最后⼀个⻓任务结束时间。如果未找到⻓任务，则与`FCP`相同）

#### 通过`performance api` 进⾏计算

通过 `performance api` ，不仅可以计算⽹络相关耗时，也可以计算性能指标，⽐如⽩屏时间`FP`、`FCP`、`LCP`、`FID`、`CLS`、`TTFB`等。

`Performance.timing `被弃⽤后，可通过 `PerformanceObserver `来获取性能指标，具体计算如下：

使⽤ `performance.getEntries `等⽅法获取性能信息的前提是对应的` Performance Entry `已

经产⽣，如对应的资源加载已经完成、⻚⾯已经完成渲染等，在加载资源前是获取不到任

何其加载的性能信息的。

为了能够判断相应的时机，可以⽤浏览器提供的 `PerformanceObserver` 来监听`Performance`

Entry 相关的事件。

##### ⽩屏时间 `FP`

⽩屏时间` FP（First Paint）`指的是从⽤⼾输⼊`url`开始到⻚⾯有内容展现出来的时间节点。（这个过程

包括`DNS`查询、`TCP`连接、`HTTP`请求、返回` HTML`、解析`HTML`）

`FP `通过 `PerformanceObserver` 实例对象中监听名为` first-paint `的 `paint` 条⽬获取，

具体代码如下：

```
const entryHandler = (list) => {

	for(const entry of list.getEntries()){

        if(entry.name === 'first-paint') {

        	observer.disconnect()

        }

        *//* *⽩屏时间*

        let FP = entry.startTime;

    }

}

const observer = new PerformanceObserver(entryHandler)

observer.observe({type: 'paint', buffered: true})
```

##### ⾸次内容绘制 `FCP`

`FCP` 通过 `PerformanceObserver `实例对象中监听名` first-contentful-paint`的

`paint `条⽬获取，具体代码如下：

```
const entryHandler = (list) => {

    for(const entry of list.getEntries()){

        if(entry.name === 'first-contentful-paint') {

        	observer.disconnect()

        }

        *//* *⾸次内容绘制时间*

        let FCP = entry.startTime;

    }

}

const observer = new PerformanceObserver(entryHandler)

observer.observe({type: 'paint', buffered: true})
```

##### 最⼤内容绘制 `LCP`

`LCP `通过 `PerformanceObserver` 实例对象的 `largest-contentful-paint `条⽬获取，

具体代码如下：

```
const entryHandler = (list) => {

    for(const entry of list.getEntries()){

        observer.disconnect()

        *//* *最⼤内容绘制*

        let LCP = entry.startTime

    }

}

const observer = new PerformanceObserver(entryHandler);

observer.observe({ type: 'largest-contentful-paint', buffered: true });
```

# 问题分析定位⼯具

## `Network` ⾯板(⽹络请求问题)

`Network` ⾯板主要是⽤来分析定位⽹络请求问题，其⾯板主要有“过滤、概览、查看详情”3个功能区

域。

![项目8](.\images\项目8.png)

`Network`⾯板可查看的基本信息有：

- 请求的`name`名称、`size`⼤⼩、`method`⽅法、`status`状态码、`scheme`协议类型、`protocol`协议版

本、`type`类型、`initiator`调⽤堆栈等

- 请求的响应时⻓
- 请求的瀑布图`waterfall`
  - `nQueued at` 浏览器将资源放⼊队列时间
  - `Started at` 浏览器开始处理资源的时间
  - `Queueing `资源放⼊队列的等待处理时间（如果遇到⾼优先级或请求超过并发限制，需要等待）
  - `Stalled `因放⼊队列⽽发⽣的停滞时间
  - `Request sent `请求发送的时间
  - `Waiting for server response `等待服务器端返回数据的时间（`TTFB`）
  - `Content Download` 浏览器下载资源的时间

`Performance`⾯板，主要分析定位⻚⾯渲染性能问题（`HTML`解析、`JS`执⾏、`CSS`解析布局计算重排、
